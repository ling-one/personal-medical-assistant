## 产品概述

彻底重构"个人医疗助手"项目架构，采用 Python 3.13 最新最佳实践，修复所有兼容性错误，删除废弃的 debug/ 目录，统一使用 Langfuse 追踪服务。

## 核心问题

1. **Python 3.13 兼容性错误（导致项目无法启动）**：

- `server/config.py`：缺少 `log_level` 字段，`.env` 中 `LOG_LEVEL=INFO` 导致 pydantic 验证失败
- `server/models/profile.py` Line 2：`from datetime import date, datetime optional` 语法错误
- `server/agent/state.py` Line 2：`Typ edDict` 拼写错误（应为 `TypedDict`）
- `server/agent/graph.py` Line 3：`Liter al` 拼写错误（应为 `Literal`）；Line 72：`add_edge(node "respond")` 缺少逗号
- `server/api/knowledge.py` Line 39：字典字面量缺少逗号

2. **过时代码（Python 3.13 已弃用）**：

- 多处使用 `Dict`、`List`、`Optional`（应改为 `dict`、`list`、`str | None`）
- `OrderedDict` 使用（Python 3.7+ 可用普通 `dict`）
- `pyproject.toml` 中 `requires-python = ">=3.11"` 未更新到 3.13

3. **架构问题**：

- `debug/` 目录完全废弃（已接入 Langfuse），应删除
- `requirements.txt` 与 `pyproject.toml` 依赖重复
- `ragflow_service.py` 疑似未使用

## 重构目标

- 修复所有 Python 3.13 兼容性错误，确保项目能正常启动
- 全面移除已弃用的 `typing` 导入（`Dict`、`List`、`Optional`）
- 删除 `debug/` 目录，完全依赖 Langfuse
- 统一依赖管理，更新 `pyproject.toml` 适配 Python 3.13
- 确保代码符合 basedpyright 严格检查（已配置 `reportExplicitAny = false` 等）

## 实施方法

### 1. 修复致命错误（项目启动阻断问题）

按依赖顺序修复，确保每一步修复后项目能进一步导入：

**Step 1: `server/models/profile.py`**

- 修复 Line 2 语法错误：`from datetime import date, datetime optional` → `from datetime import date, datetime`
- 添加 `from datetime import datetime` 用于 `created_at`/`updated_at` 字段

**Step 2: `server/agent/state.py`**

- 修复 Line 2：`from typing import TypedDict, Any` → 修正拼写为 `TypedDict`
- 确认 `total=False` 拼写正确

**Step 3: `server/config.py`**

- 添加 `log_level: str = "INFO"` 字段（匹配 `.env` 中的 `LOG_LEVEL=INFO`）
- 添加 `log_file: str = "logs/app.log"` 字段（`.env` 中未设置但有 `DEBUG=false`）

**Step 4: `server/agent/graph.py`**

- 修复 Line 3：`from typing import Literal` → 修正拼写为 `Literal`
- 修复 Line 72：`graph.add_edge(node "respond")` → `graph.add_edge(node, "respond")`

**Step 5: `server/api/knowledge.py`**

- 修复 Line 39：`{"document_id": doc_id, "status": "added"}` → 添加缺失逗号

### 2. 移除过时代码

批量替换以下模式（使用 `basedpyright` 警告驱动）：

- `from typing import Dict, List, Optional` → 移除，改用内置 `dict`、`list`、`str | None`
- `OrderedDict` → `dict`（Python 3.7+ 保证插入顺序）
- `Dict[str, Any]` → `dict[str, Any]`
- `List[str]` → `list[str]`
- `Optional[str]` → `str | None`

涉及文件：

- `server/models/conversation.py`
- `server/models/profile.py`
- `server/services/memory_service.py`
- `server/services/feedback_service.py`
- `server/agent/state.py`

### 3. 删除废弃目录

- 删除 `debug/` 整个目录（已接入 Langfuse，不再需要自建追踪）
- 从 `server/services/__init__.py` 移除 `trace_service` 的导入检查（保留 `trace_service.py` 本身，因为它已改为 Langfuse 实现）

### 4. 统一依赖管理

- 保留 `pyproject.toml` 作为权威依赖声明（Poetry 格式）
- 更新 `requires-python = ">=3.13"`
- 更新 `ruff` 的 `target-version = "py313"`
- 可选：生成新的 `requirements.txt`（通过 `poetry export`）

### 5. 验证

- 每一步修复后运行 `python -c "import server.config"` 验证
- 最终运行 `basedpyright` 检查剩余错误
- 运行 `python -m uvicorn server.main:app --port 8000` 验证启动

## 性能考量

- 本地向量模型 `BAAI/bge-m3` 首次加载较慢，可在服务启动时预加载
- Langfuse 追踪为异步操作，不阻塞主请求流程
- 使用 `basedpyright` 的增量检查（`--project`）加速类型检查

## 避免技术债

- 不使用 `typing` 中的 `Dict`、`List`、`Optional`（Python 3.9+ 已弃用）
- 不保留 `debug/` 目录（避免维护两套追踪系统）
- 不混合 `requirements.txt` 和 `pyproject.toml` 的依赖声明
